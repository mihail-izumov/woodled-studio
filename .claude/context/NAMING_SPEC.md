# Спецификация названий светильников — WOODLED ROTOR

> Как формируется подпись светильника на **странице комнаты**, в **навбаре** страницы светильника и в **плашке** карточки.
> 17 моделей · 5 типов · 4 зоны · линейки Rotor / Rotor X / Rotor Elliptical (+ Jupiter в планах).
> Принцип: одна модель данных → несколько сборок под разные поверхности.

---

## 1. Главный принцип

Идентичность светильника = слои, которые нельзя слеплять в одну строку (как было в «Бра Vertical L»):

| Слой | Что это | Где живёт визуально |
|---|---|---|
| **Тип** | люстра / спот / бра / торшер / настольная. Универсален для всех линеек | слово слева (комната) + заголовок (карточка) |
| **Линейка / бренд** | Rotor · Rotor X · Rotor Elliptical (внутри бренда Rotor); далее Jupiter… | **подпись снизу — всегда** |
| **Различитель** | то, чем покупатель отличает варианты внутри типа: **размер** (люстры) или **форма** (бра/торшер/спот) | чип справа |
| **Дерево** | порода (дуб/орех/чёрный) | шар-иконка |

Подпись линейки показывается **у всех типов и сразу** (решение бренда — единая подпись WOODLED). Значит каждая строка на странице комнаты — двухстрочная; карточки зон с запасом по высоте это держат.

Поверхность решает, какие слои показать — по месту и намерению:

| Поверхность | Место | Что показываем |
|---|---|---|
| **Страница комнаты** (строка, 2 колонки) | очень тесно | тип (слева) + чип-различитель (справа) + подпись-линейка снизу |
| **Навбар страницы светильника** (iOS) | тесно: слева «‹ {Комната}», справа иконка, режется | `Тип различитель`, ≤ ~15 символов (без линейки) |
| **Плашка карточки** | во всю ширину | полное `Тип различитель` + подпись линейки снизу |

---

## 2. Что несёт чип и подпись у каждого типа

Метры — не универсальная валюта. У каждого типа свой различитель.

| Тип | Чип справа (комната) | Подпись снизу |
|---|---|---|
| **Люстра** | **размер**: малая / средняя / большая / огромная | Rotor · Rotor X · Rotor Elliptical |
| **Бра** | **ориентация**: верт. / гор. | Rotor |
| **Торшер** | **форма**: тренога / стойка | Rotor |
| **Спот** | **монтаж**: подвесной / настенный | Rotor |
| **Настольная** | — | Rotor |

**Почему у люстры чип — размер, а различие моделей ушло в подпись.** Rotor, Rotor X и Rotor Elliptical — три реально разных изделия. Их не описать выдуманным словом формы («круглая/овальная» — это уже было ошибкой). Берём настоящие имена линеек и пишем их подписью снизу. Чип тогда освобождается под размер — то, чем люстры различают в голове (малая…огромная). У остальных типов линейка одна (Rotor), и она тоже стоит подписью — единая подпись бренда на всех.

## 4. Страница светильника — три элемента имени

### 4.1 Навбар (системный заголовок iOS)
```
‹ {Комната}        {Тип} {различитель}        🔇
```
Коротко: `Люстра большая`, `Бра гор.`, `Торшер тренога`, `Спот настенный`, `Настольная`. Всё ≤15 символов. Линейку в навбар не выносим.

### 4.2 Плашка (вверху страницы)
```
[шар дерева]  {Заголовок карточки}              {цена}
              {Линейка — подпись}
              [чип дерева]  [статус]
```
Полное имя (`Люстра большая`, `Бра горизонтальное`, `Торшер на треноге`) + линейка подписью снизу (`Rotor X`, `Rotor`). Артикул — в «Деталях».

### 4.3 Переключатель размера (только у люстр)
Размер словом + площадью; голую букву S/M/L/1000 не показываем. Ряд зависит от линейки:
- **Rotor**: малая · 8–10 / средняя · 10–12 / большая · 12–15 / огромная · 15–20
- **Rotor X**: средняя · 10–14 / большая · 14–18
- **Rotor Elliptical**: малая · 5–15 / большая · 15–20

У спота, бра, торшера, настольной переключателя размера нет.

## 5. Линейки, бренды и подпись

Два уровня группировки:

- **Бренд** (Rotor / Jupiter) — крупная коллекция. **Одна на комнату по умолчанию** (связность — суть продукта). Смешение брендов — редкий продвинутый режим.
- **Линейка люстр** (Rotor / Rotor X / Rotor Elliptical внутри бренда Rotor) — подсемейство люстр. Свободно сочетается с бра, торшерами и т. д. того же бренда.

**Правило подписи снизу — показываем всегда, у всех типов:**
- **Люстры** — линейка: `Rotor` / `Rotor X` / `Rotor Elliptical` (три разных изделия).
- **Остальные типы** — бренд: `Rotor`.

Это решение бренда: единая подпись WOODLED под каждым светильником, ради узнаваемости. Когда появится Jupiter — у не-люстр подпись переключается `Rotor` / `Jupiter`, у люстр — `Rotor X` / `Jupiter X` и т. д.

**Способ показа** — мелкой подписью под типом (не вторым чипом): в двух колонках строка несёт один правый токен, а «заголовок + подпись» — родной паттерн для вторичного слоя. Все строки теперь двухстрочные; это снимает и вопрос латиницы бренда — имя линейки живёт подписью, а не вклеивается в заголовок.

**Поле `collection`** — enum: `rotor`, `rotor_x`, `rotor_elliptical` (заполнены), `jupiter…` (`coming`, в пикере не светится, пока пуст). Новый бренд = дозапись строк + свой словарь различителей по типам.

## 6. Правила отображения

**Счётчик ×N — по умолчанию выкл.** Одна строка на один физический светильник; опции живут на штуке, поэтому два бра с разными лампами не `q:2`. `×N` только при полном совпадении `модель + дерево + ВСЕ опции`; любое различие — авто-расклейка. Счётчик дублируют точки внизу карточки зоны.

**Спот — без размера в комнате.** Spot S и Spot L кроют одинаково (3–5 м²), различие — диаметр абажура. На странице комнаты — просто `Спот` + монтаж (`подвесной` / `настенный`). S/L живёт только в карточке.

**Сокращения — внутри типа, не между.** Бра — сокращённо в чипе/навбаре (`верт./гор.`), полностью в плашке (`вертикальное/горизонтальное`). Люстра/торшер/спот — полными словами. Люстра и бра в разных карточках зон, рядом не стоят — рассинхрон не читается.

**Коллизия (редко).** Две люстры в одной зоне физически невозможны (лимит 1); для бра/спота вторая ось включается лишь при реальном совпадении. По умолчанию не показываем.

## 7. Обоснование ключевых решений

**Люстра — линейка подписью, размер чипом.** Rotor / Rotor X / Rotor Elliptical — три разных изделия; их имена и есть различитель, выдуманные слова формы не нужны. Размер (то, чем люстры меряют в голове) встаёт чипом справа.

**Спот и Юнит — два разных спота.** Spot — длинный подвесной (потолок); Unit — компактный настенный (стена). Названы по монтажу; размер у спота не показываем.

**Торшер — тренога / стойка.** Два силуэта, а не «большой и копия». Слово-форму получают оба; «S» не используем.

**Заголовок — три регистра, не «простор».** Навбар iOS зажат кнопкой комнаты и режется → короткий `Тип различитель`. Полное имя + линейка — только в плашке.

**Линейка — подписью снизу, у всех и сразу.** В двух колонках строка несёт один правый токен; линейка — вторичный слой, её родной паттерн «заголовок + подпись». Единая подпись бренда под каждым светильником — осознанное решение ради узнаваемости WOODLED.

## 8. Модель данных

| Поле | Значение / назначение |
|---|---|
| `type` | люстра/спот/бра/торшер/настольная — слово слева + бейдж |
| `zone` | Потолок / Стены / Пол / Мебель |
| `collection` | `rotor` / `rotor_x` / `rotor_elliptical` (сейчас); `jupiter…` (`coming`) |
| `discriminator` | `size` (люстры) или `form` (бра/торшер/спот) или `none` |
| `chipLabel` | чип справа (комната) + слово в навбаре: размер у люстр, форма у остальных |
| `titleFull` | полная форма в плашке (`горизонтальное`, `на треноге`) |
| `sizeRow` | ряд переключателя размера (только у люстр) |

**Производные подписи (движок собирает сам):**
- Строка комнаты = `шар(дерево)` + `type` (слева) + `chipLabel` (чип справа) + подпись `collection` снизу.
- Навбар = `type` + ` ` + `chipLabel`.
- Заголовок карточки = `type` + ` ` + `titleFull` + подпись `collection` снизу.
- Переключатель = `sizeRow` (слово + площадь), только у люстр.

`collection` для подписи: `rotor`→«Rotor», `rotor_x`→«Rotor X», `rotor_elliptical`→«Rotor Elliptical».
