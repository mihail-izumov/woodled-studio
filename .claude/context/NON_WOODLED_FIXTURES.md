# Пользовательские светильники в конфигураторе

> Версия: 1.0 | Дата: 5 июня 2026
> Контекст: конфигуратор дома WOODLED — карточки зон на экране комнаты
> Связанные документы: `WOODLED_tone_of_voice.md`, `TEMPLATES_SYSTEM.md`

---

## 1. Зачем пользовательские светильники

В реальном доме WOODLED — не единственный свет. Люстра Rotor висит над столом, но на тумбочке стоит IKEA, а в коридоре — спот из Леруа. Конфигуратор считает свет на весь дом, поэтому ему нужны все источники, не только наши.

WOODLED при этом остаётся «вишенкой на торте» — дорогой, деревянный, штучный. Задача UI: показать это без слов «наш» и «чужой».

---

## 2. Проблема: ярлык «СВОЙ»

В первой версии бейдж под названием пользовательского светильника показывал `СВОЙ` — в том же месте, где у WOODLED стоит `ROTOR`.

Почему это плохо:

- **Создаёт бинарность.** `ROTOR` vs `СВОЙ` — пользователь читает как «наш» vs «не наш». Это ставит его торшер из IKEA в позицию второго сорта.
- **Не по ToV.** «Свой» — непонятное слово. Свой чей? Пользовательский? Самодельный? Звучит как техническая заглушка, не как осознанный выбор.
- **Называет отсутствие.** Любой ярлык вместо бренда («сторонний», «другой», «не WOODLED», «без бренда») привлекает внимание к тому, чего нет. Это антипаттерн для премиального продукта.

---

## 3. Принцип: не называй отсутствие, наполняй присутствие

Референс — Tesla. В приложении Tesla:
- Supercharger: бренд, статус зелёный/красный, время ожидания, анимация, кол-во стоек
- Чужая зарядка: пин на карте, тип разъёма (CCS / CHAdeMO), мощность

Tesla не пишет «NOT TESLA» на чужих зарядках. Она делает свои богаче — и разница считывается через **плотность информации**, а не через ярлык.

Перенос на WOODLED: карточка WOODLED уже насыщена (точка дерева, бейдж ROTOR, дескриптор модели). Пользовательской карточке не нужен «антибренд» — ей нужно просто не иметь этих элементов.

---

## 4. Решение: три слоя

### Слой 1 — Убрать «СВОЙ», ничем не заменяя

Где у WOODLED стоит строка `ROTOR` зелёным шрифтом — у пользовательского светильника этой строки нет. Карточка становится визуально легче. Глаз сам считывает: вот карточка с идентичностью, а вот карточка-функция.

### Слой 2 — Визуальная разница через плотность

| Элемент | WOODLED | Пользовательский |
|---|---|---|
| Цветная точка | Тёплая (дуб / орех / чёрный) | Нейтральная серая |
| Бренд-строка | `ROTOR` · `T.brand` | Строки нет |
| Дескриптор | большая, верт., тренога | Можно: `~3000 лм` |
| Визуальный вес карточки | Полный | Облегчённый |

Пользователь не чувствует «второй сорт». Карточка просто проще — она говорит: «я тут для расчёта, и всё». А WOODLED-карточка говорит: «я — дерево, модель, история».

### Слой 3 — Копирайт при добавлении (по ToV)

Когда пользователь добавляет не-WOODLED светильник, тост:

> Есть. Укажите примерную яркость — остальное посчитаем.

Или:

> Добавили. Для расчёта нужны люмены — модель и бренд не важны.

Принцип: мы считаем свет, а не бренды. Ноль пафоса, ноль сравнений.

---

## 5. Что НЕ делать

| Вариант | Почему нет |
|---|---|
| `СВОЙ` | Непонятно, звучит как баг |
| `ДРУГОЙ` | Противопоставление — «другой» = «не наш» |
| `БЕЗ БРЕНДА` | Акцент на отсутствии, снисходительно |
| `СТОРОННИЙ` | Канцелярит, не ToV |
| `ПОЛЬЗОВАТЕЛЬСКИЙ` | Длинно, технично, не говорят так за столом |
| Иконка ≠ вместо точки | Привлекает внимание к различию |
| Серый текст `не WOODLED` | Буквально называет отсутствие бренда |

## 6. Поведение в UI

### Карточка в зоне (ZoneCard)

```
WOODLED:                          Пользовательский:
┌─────────────────────┐           ┌─────────────────────┐
│ ● Люстра            │           │ ● Торшер            │
│   ROTOR    большая  │           │                     │
└─────────────────────┘           └─────────────────────┘
  ↑ тёплая точка                    ↑ серая точка
  ↑ бренд + дескриптор              ↑ только название типа
```

### Добавление светильника

При добавлении пользовательского светильника — запрашиваем:
1. **Тип** (люстра / бра / торшер / настольная / спот) — выбор из списка
2. **Яркость** (люмены) — поле ввода или ползунок с подсказками типовых значений

Не запрашиваем: бренд, модель, цвет, температуру. Это лишнее для расчёта и перегружает UX.

### Подсказки яркости (помощь при заполнении)

| Подсказка | Люмены |
|---|---|
| Ночник, свеча | ~200 лм |
| Настольная лампа | ~400–600 лм |
| Средний торшер | ~800–1200 лм |
| Яркая люстра | ~2000–3000 лм |
| Очень яркая люстра | ~4000–5000 лм |

Формулировка в UI: «Не знаете точно — выберите примерно. Для расчёта этого хватит.»

## 7. Эффект на бренд

Тихая, визуальная разница между карточками делает три вещи:

1. **Учит видеть WOODLED.** Пользователь быстро запоминает: тёплая точка + ROTOR + дескриптор = WOODLED. Серая точка без бейджа = всё остальное. Бренд узнаётся по присутствию, а не по сравнению.

2. **Не обижает.** Ничей торшер не назван «чужим» или «своим». Он просто функциональный элемент расчёта. Это уважительно и совпадает с ToV (спокойный, тёплый, по делу).

3. **Продаёт через контраст.** Когда пользователь видит рядом богатую WOODLED-карточку и простую серую — он сам чувствует разницу. Не потому что мы сказали «мы лучше», а потому что мы *показали* больше. Это и есть принцип Tesla: наполняй своё, не комментируй чужое.
