# `.claude/context/` — База знаний WOODLED для AI-ассистента

Контекст, который AI-ассистент должен прочитать **в начале сессии** (после `CLAUDE.md` в корне). Здесь — спеки, словари, голос и рационал. Алгоритмы и пороги меняются, эти документы — источник правды.

## Правила для AI

1. **Сначала** прочитай `CLAUDE.md` в корне — там карта проекта, конвенции, подводные камни.
2. **Затем** прочитай этот README и нужные документы из таблиц выше — в зависимости от задачи.
3. Перед правкой текстов в карточках / легенде / Story — обязательно прогон по чек-листу ToV (в `WOODLED_словарь_карточек_настроения.md` секция «✅ Чек-лист по ToV») и проверка по `WOODLED_tone_of_voice.md`.
4. **Никогда** не используй в UI-текстах `collection` и `name` модели («Rotor», «Rotor 1000»). Только `MD[m].type` («люстра», «бра», «торшер») + `MD[m].chip` («огромная», «большая»). См. `NAMING_SPEC.md`.
5. **Пороги** в `BRIGHT` (moods.ts), `lightState` (copy.ts) и `brightLevel` (forest.ts, forest-cards.ts) должны быть синхронны: `0.5 / 0.8 / 1.2 / 1.5`.
6. **Падежи** в фразах карточек: «у/от/без X» → `fxNameGen` + `FURN_GEN`; «к X» → `fxNameDat`; везде иначе — `fxNameNom`.

## Как давать задачи

Называй конкретный экран/компонент, прикладывай скриншот, формулируй цель (а не только пиксели). Прототипы дизайна — отдельным `.jsx`-артефактом, потом перенос в `.vue`.
