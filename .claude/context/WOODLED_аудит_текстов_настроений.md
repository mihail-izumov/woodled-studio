# WOODLED — аудит текстов настроений (чек-лист)

> Цель: проверять тексты блока настроения — **имя сцены**, **легенда** и **карточки «Как это работает»** — на пороги, триггеры, грамматику и Tone of Voice. Документ — памятка перед сложными правками. Полный словарь фраз — в `WOODLED_словарь_карточек_настроения.md`.
>
> Контекст: настроение комнаты = **лесная сцена** (имя = `порода × место`). Дашборд — «сколько света»; настроение — «какой и откуда». Тексты собираются детерминированно: движок выбирает и склеивает одобренные фрагменты, не сочиняет.

## 2. Поток данных (комната → текст)

```
room (fixtures, furniture, ceilingH, wallFinish, typeId, sizeIndex)
   │
   ├─ fxLm(fixtures)            сумма света (с body/ambient/диффузором)
   ├─ baseLm(rt, room)          норма (lux·площадь·мебель / (UF·MF))
   ├─ zoneLm(fixtures, zone)    люмены по зонам
   ├─ furnPct(furniture)        % потерь на мебели
   ▼
forest.ts:
   forestScene(rt, room) → { name, legend, place, wood }
forest-cards.ts:
   assembleCards(rt, room)  → KnobCard[]  (filtered — пустые карточки скрываются)
   ▼
RoomDetail.vue (computed scene, knobs) → ForestMood.vue (рендер)
```

Реактивность: `scene`/`knobs` — `computed` в `RoomDetail`. Меняешь набор/мебель/тепло/стены/потолок → пересчёт → `ForestMood` перерисовывается. Если считать внутри `ForestMood` от `props.room` — глубокая реактивность теряется (так уже было).

## 3. Имя сцены = ПОРОДА + МЕСТО

`name = ${PORODA[wood]} ${PLACE_RU[place]}` (`forestScene` в `forest.ts`).

**ПОРОДА** ← доминирующее дерево (`woodOrder`): больше всего точек (по `q`); при равенстве — по вкладу люмен.

| Wood | PORODA |
|---|---|
| oak | Дубовая |
| walnut | Ореховая |
| black | Вековая |

**МЕСТО** ← `scenePlace` по **вертикали света**: `ceilingShare = zoneLm(ceiling) / fxLm`.

| Условие | place | PLACE_RU |
|---|---|---|
| ceilingShare ≥ `CEILING_HIGH` (0.6) | glade | поляна (свет сверху, открыто) |
| ceilingShare ≤ `CEILING_LOW` (0.25) | thicket | чаща (свет внизу) |
| иначе | grove | роща (свет ровно) |
| total ≤ 0 | grove | — (запасной) |

Проверки имени:
- только потолок (ceilingShare=1) → `…поляна`; только бра/торшер (ceilingShare=0) → `…чаща`; смешанно → `…роща`.
- доминирует чёрный дуб → `Вековая …`.
- ⚠ Место — про РАСПРЕДЕЛЕНИЕ, не яркость. Тусклый верхний свет — всё равно поляна. Имя НЕ должно зависеть от `ratio`.

## 4. Легенда

Сборка (`forestScene`): `${LEAD[place]}, ${BRIGHT_WARM[bright][warmth]}. ${Заглавная(woodPhrase)}, ${furnPhrase(furnPct)}.`

**LEAD (место — должно совпадать со `scenePlace`!):**

| place | LEAD |
|---|---|
| glade (поляна) | «Главный свет идёт сверху» |
| grove (роща) | «Свет ровно по всей комнате» |
| thicket (чаща) | «Свет собрался внизу» |

**Яркость** (`brightLevel(ratio)` в `forest.ts`): `<0.8` → dim · `0.8–1.2` → norm · `≥1.2` → full.
**Синхронизация:** пороги совпадают с `BRIGHT` (`moods.ts`) и `lightState` (`copy.ts`). Раньше было 0.8/1.4 — устарело.

**Тепло** (`tempWarmth` — доминанта `btemp` по люменам): `2700` → warm · `3000` → neutral · `4000`/прочее → clear. Дефолт `btemp='4000'`.

**BRIGHT_WARM[bright][warmth]** (9 связок) — таблица в коде. Напр.: norm+clear = «дневной и ясный», dim+warm = «его немного, тёплый и мягкий», full+clear = «его вдоволь, ясный».

**Дерево** (`woodPhrase`): 1 порода → `WOOD_NOM` («дуб»/«орех»/«чёрный дуб»); 2+ → `${WOOD_NOM[доминанта]} с ${WOOD_INS[вторичное]}` («орех с дубом»). `WOOD_INS` — творительный: дубом/орехом/чёрным дубом.

**Плотность** (`furnPhrase(furnPct)`): `≤0` → «свету ничего не мешает» · `≤12` → «мебели немного» · `>12` → «у пола держится мягкая тень».

Проверки легенды:
- первое предложение (LEAD) ДОЛЖНО соответствовать месту в имени (иначе баг: имя «…поляна», а легенда «свет собрался внизу»);
- яркостно-тепловая связка — реальному `ratio` и доминантной температуре;
- `woodPhrase` — доминанте дерева и склонению в смеси;
- заглавная буква во втором предложении (`cap`).

## 5. Карточки «Как это работает» (`assembleCards` в `forest-cards.ts`)

7 карточек **с правилом скрытия** — если данных нет, карточка не показывается.

| # | title | chip | text зависит от | Скрывается, если |
|---|---|---|---|---|
| 1 | Где свет | доминантная зона + `%` («потолок 55%») | `scenePlace` + список вторичных моделей | — (всегда показ при наличии светильников) |
| 2 | Сколько света | `round(ratio·100)%` | `brightLevel` + `room.typeId` (per-room обучение) | — |
| 3 | Стены | «светлые» / «тёмные» | `room.wallFinish` | `wallFinish === 'medium'` |
| 4 | Дерево | состав со склонениями (`6 орехов · 3 дуба`) | `woodOrder` + `WOOD_CHAR` | — |
| 5 | Мебель | `−${furnPct}%` или «зеркало» | топ-2 предмета + `FURN_HINT` совет | `furniture.length === 0` |
| 6 | Оттенок света | одна температура или смесь (`2700K · 4000K`) | `tempWarmth` или группировка по `btemp` | — |
| 7 | Потолок и подвес | `${h} м[ · подвес]` | `ceilingH` + `hasCeilingPendant` | `ceilingH ≈ 2.7` И нет подвеса |

**Падежи в фразах:**
- `fxNameNom(modelId, count)` — именительный («огромная люстра», «два торшера», «пара бра»).
- `fxNameGen(...)` — родительный («огромной люстры», «двух торшеров») — для «от/у/без X».
- `fxNameDat(...)` — дательный («огромной люстре», «двум торшерам», «паре бра») — для «к X».
- `FURN_GEN[id]` — родительный имени мебели («дивана», «стеллажа», «кровати») для «у X».
- `zoneKindNom(zone, n)` — для совета по зоне («торшер» / «два торшера» / «N торшеров»).

**Названия моделей в текстах:**
- **Никогда** не использовать `collection` («Rotor», «Rotor X», «Rotor Elliptical») и `name` («Rotor 1000»).
- **Только** `MD[m].type` (`люстра`/`спот`/`бра`/`торшер`/`настольная`) + `MD[m].chip` (размер — для люстр).
- См. `NAMING_SPEC.md`.

## 6. Как ловить ошибки (метод)

1. **Имя ↔ легенда ↔ карточка «Где свет».** Место одно и то же во всех трёх. Имя «…чаща» + легенда «Главный свет идёт сверху» = баг.
2. **Пороги (граничные значения).** Проверять ровно на границах: `ceilingShare` = 0.6 и 0.25; `ratio` = 0.8 и 1.2; `furnPct` = 5/12; `ceilingH` = 2.7; одна vs ≥2 температуры (mixed).
3. **Триггер → текст вручную.** Для тестовой комнаты посчитать `ceilingShare`, `ratio`, доминанту дерева/температуры, `furnPct` — и сверить с выводом движка.
4. **Реактивность.** Изменить светильник / мебель / тепло / стены / потолок → имя, легенда и карточки обновились вживую (см. §2).
5. **Грамматика и падежи.** «У» / «от» / «без» → родительный (`fxNameGen`, `FURN_GEN`). «К» → дательный (`fxNameDat`). Глагол согласован с числом (`compensируют`/`компенсирует`).
6. **Склонения дерева.** `woodWord(wood, count)` — 1 «дуб» / 2-4 «дуба» / 5+ «дубов». Проверять граничные количества (1, 2, 5, 11, 21).
7. **Чипы.** Числа округлены; проценты — от нормы; знак «−» у мебели; формат «6 орехов · 3 дуба» (с « · »).
8. **Tone of Voice.** «Вы», без AI-штампов и сравнений «как…», тире по делу. Сверять с `WOODLED_tone_of_voice.md` и чек-листом в `WOODLED_словарь_карточек_настроения.md`. Особое внимание к состоянию «мало/темно» — только забота, без юмора.
9. **Внутренние коэффициенты наружу не текут.** В текстах/чипах не должно быть сырых `body`/`ambient`/`×0.65`/`UF=0.55`/«ratio 0.66».
10. **Скрытие карточек.** Проверять, что карточка реально скрыта при пустых данных (Стены при `medium`, Мебель при пустой, Потолок при стандартном 2.7 без подвеса).
11. **Edge cases:** пустая комната (0 светильников) → `EMPTY_SCENE` (имя «—»); одна зона; смешанные деревья и температуры; `furnPct = 0`; зеркало (`ab < 0` → отдельная фраза «возвращает 3%»).

## 7. Тест-набор (эталонные выводы для сверки)

| Комната (набор) | Имя | Легенда |
|---|---|---|
| Гостиная: огромная люстра (потолок, 55%) + 2 бра + 2 торшера + настольная; орех+дуб; 4000K (торшеры 2700K); мебель −20%; потолок 3 м, подвес | **Ореховая роща** | «Свет ровно по всей комнате, дневной и ясный. Орех с дубом, у пола держится мягкая тень.» |
| Только средняя люстра; дуб; 2700K | **Дубовая поляна** | «Главный свет идёт сверху, его немного, тёплый и мягкий. Дуб, свету ничего не мешает.» |
| Бра + торшер; дуб; 2700K; потолок выкл (0%) | **Дубовая чаща** | «Свет собрался внизу, его немного, тёплый и мягкий. Дуб, у пола держится мягкая тень.» |
| Только люстра; чёрный дуб; 4000K | **Вековая поляна** | «Главный свет идёт сверху, дневной и ясный. Чёрный дуб, свету ничего не мешает.» |

Карточки гостиной-эталона (со скрытыми): `[потолок 55%] Где свет` · `[105%] Сколько света` · ~~`Стены`~~ (medium) · `[3 ореха · 3 дуба] Дерево` · `[−20%] Мебель` · `[2700K · 4000K] Оттенок света` · `[3 м · подвес] Потолок и подвес`. Итого **6 карточек**.

## 8. Цвет (для проверки заодно)

- Акцент блока и чипов = **цвет комнаты** `room.cardColor`. Поток: `RoomDetail` (`tint = room.cardColor ?? T.neutral`) → `ForestMood` проп `tint` → рамка блока (теперь прозрачная — без border), фон карточек, чипы, активная точка, иконка кнопки.
- Старый `mood.color` для тонировки больше не используется.
- Проверить: все акценты в `ForestMood.vue` берутся из `props.tint`, нет захардкоженных цветов (кроме `T.text`/`T.textSec`/`T.bg`/`T.card` из токенов).

## 9. Оговорки

- Песочница без `node_modules` — собрать/типчекнуть нельзя. Проверять: node-репликой формул либо `npm run docs:dev`.
- `forest.ts` — источник правды по имени и легенде. `forest-cards.ts` — источник правды по карточкам. Правка текстов = правка таблиц-банков в них, логику не трогать.
- Связанные доки: `WOODLED_словарь_карточек_настроения.md` (полный словарь фраз карточек + чек-лист ToV), `WOODLED_сцены_настроения_спека.md` (концепция мест и пород), `WOODLED_tone_of_voice.md` (голос), `WOODLED_рационал_и_архитектура.md` + `WOODLED_коэффициенты_светильников.md` (почему нормы и факторы такие), `WOODLED_шкала_яркости_BRIGHT.md` (шкала бейджа).
