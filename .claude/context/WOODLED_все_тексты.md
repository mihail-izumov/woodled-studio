# WOODLED — банки текстов дашборда и мебели

Тексты для подсказок дашборда (`engine/copy.ts`), мебели (`data/furniture.ts`), бейджа яркости и статичных подписей UI.

> 📍 **Для блока настроения** (имя сцены, легенда, 7 карточек) — отдельный файл `WOODLED_словарь_карточек_настроения.md`. Здесь — всё, что вокруг.
> 
> Раздел про старые «Утро в лесу / Ясный полдень / Тёплые сумерки» (`MOODS`, `moodPhase`, `moodDesc`, MoodDetailModal) **удалён**: модуль мёртвый, в UI не появляется. Тип/имя места теперь — `forestScene` в `engine/forest.ts`.

Принципы:
1. Живая разговорная речь, законченные фразы (без обрывов).
2. Поменьше повелительного «-йте» — лучше безлично/инфинитив («просится бра», «можно приглушить»).
3. Не клинические ярлыки — человеческие описания.
4. Без AI-оборотов и сравнений «как…».
5. Мысль договорена.

Подстановки: `{pct}` — % от нормы, `{z}` — пустая зона, `{name}` — предмет, `{zone}` — зона.
Несколько вариантов на состояние = движок крутит их по очереди (анти-повтор по сиду).

# A. Динамические тексты

## A1. Подсказка на дашборде (`copy.ts/lightHint`) — варианты на состояние

> Без точки в конце (фраза живёт в пилюле). При «светло / ярко / праздник» подсказка НЕ повторяет бейдж яркости (A3), а советует, что делать.

**≤ 0.5 (темно), зона известна:**
- Пока темно — {z} пригодится
- Света совсем мало — {z} просится

**≤ 0.5, зоны нет:**
- Пока темно — света маловато
- Темновато, стоит добавить света

**≤ 0.8 (полусвет), зона известна:**
- Света на {pct}% от нормы — {z} не помешает
- Света немного не хватает — {z} просится

**≤ 0.8, зоны нет:**
- Света на {pct}% от нормы — можно чуть добавить
- Немного не дотягивает до нормы — есть куда расти

**≤ 1.2 (светло) — совет, не повтор бейджа:**
- Менять ничего не нужно
- Вот так хорошо
- Можно оставить как есть

**≤ 1.5 (ярко):**
- При желании можно приглушить
- Ярковато — можно убавить

**> 1.5 (праздник):**
- Глазам столько не нужно — можно приглушить или добавить диммер
- Стоит снять пару светильников или поставить диммер

# C. Расширенный банк (композиция) — как получаем сотни фраз

Фраза дашборда собирается из трёх кусочков: **зачин состояния** + **фрагмент по комнате** + **связка**. Вручную пишем ~30 кусочков — на выходе сотни комбинаций, и все звучат живо.

## C1. Зачины по состоянию (только «темно/полусвет» — где собираем из кусочков)
| состояние | варианты зачина |
|---|---|
| ≤ 0.5 темно | Пока темно · Света совсем мало · Темновато |
| ≤ 0.8 полусвет | Света на {pct}% от нормы · Света немного не хватает · Немного не дотягивает до нормы |

> Состояния «светло / ярко / праздник» НЕ собираются из кусочков — у них готовые фразы-советы (см. A1), потому что бейдж яркости уже говорит состояние, а подсказка должна советовать.

## C2. Фрагменты по комнате (именительный, подлежащее) — таблица `ROOM_FRAGMENT` в `copy.ts`
| комната | фрагмент |
|---|---|
| Кухня | пара спотов над столешницей |
| Спальня | бра у изголовья |
| Гостиная | торшер у дивана |
| Кабинет | лампа на рабочем столе |
| Детская | лампа на столе |
| Коридор | пара спотов вдоль прохода |
| Ванная | свет у зеркала |
| Лестница | бра вдоль ступеней |

## C3. Связки (универсальные — не про способ монтажа: бра вешают, торшер ставят, спот монтируют)
не помешает · пригодится · просится · напрашивается

## C4. Правило сборки (без точки в конце — фраза в пилюле)
- **Темно / полусвет:** `{зачин} — {фрагмент} {связка}` (собирается из кусочков).
- **Светло / ярко / праздник:** готовая фраза-совет из A1 (не из кусочков, бейдж уже сказал состояние).

## C5. Примеры сборки (что увидит пользователь)
| комната | темно | полусвет | светло | праздник |
|---|---|---|---|---|
| Кухня | Пока темно — пара спотов над столешницей не помешает | Света на 60% — пара спотов над столешницей пригодится | Менять ничего не нужно | Глазам столько не нужно — можно приглушить или добавить диммер |
| Спальня | Темновато — бра у изголовья просится | Света немного не хватает — бра у изголовья напрашивается | Вот так хорошо | Стоит снять пару светильников или поставить диммер |
| Кабинет | Пока темно — лампа на рабочем столе пригодится | Немного не дотягивает до нормы — лампа на рабочем столе не помешает | Можно оставить как есть | При желании можно приглушить |
| Ванная | Пока темно — свет у зеркала не помешает | Света на 70% — свет у зеркала пригодится | Менять ничего не нужно | Ярковато — можно убавить |

## C6. Реакция на последнее действие (`actionReaction` в `copy.ts`)
Короткая подпись на ~1 сек после изменения. 2 варианта на триггер.

| что сделал | варианты |
|---|---|
| добавил светильник | Стало светлее · Прибавили свет |
| убрал светильник | Стало темнее · Света поубавилось |
| прибавил патроны | Каждый патрон добавил света · Ярче |
| поднял потолок | Потолок выше — свету идти дальше · Высокому потолку нужно больше света |
| опустил потолок | Потолок ниже — света хватает легче |
| стены темнее | Тёмные стены забрали часть света |
| стены светлее | Светлые стены вернули часть света |
| выбрал модель ярче | Света прибавилось |
| выбрал модель мягче | Стало мягче |

## C7. Сколько в итоге сценариев
- Дашборд: ~150+ (C1×C2×C3 + варианты).
- Реакции на действие: ~18 (C6).
- Карточки настроения + легенда: см. `WOODLED_словарь_карточек_настроения.md`.
- Тосты мебели: 4 шаблона × 4 рода.
- **Итого банк** — несколько сотен уникальных фраз из ~50–60 написанных кусочков. Движки `copy.ts` и `forest-cards.ts` собирают и не повторяются (анти-повтор по сиду).
