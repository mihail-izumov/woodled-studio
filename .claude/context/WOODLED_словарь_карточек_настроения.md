# WOODLED — словарь карточек блока настроения

Все тексты для 7 карточек «Как это работает» в блоке настроения комнаты.
Применён ToV (`WOODLED_tone_of_voice.md`). Каждая фраза — на «вы», коротко, глаголами, с конкретикой, без AI-штампов. Карточки прячутся, если данных нет.

Под капотом — банки фрагментов, движок крутит варианты по сиду `fixtures.length * 17 + furnPct(furniture)` (анти-повтор). Каждый «слот» имеет 2–3 синонимичных формулировки.

---

## 0. Источники данных

| Откуда | Что используем |
|---|---|
| `MD[m].type` | «люстра» / «спот» / «бра» / «торшер» / «настольная» |
| `MD[m].chip` | размер: «малая» / «средняя» / «большая» / «огромная» |
| `MD[m].lamps × q × f.l` | количество ламп |
| `f.opts.btemp` | температура `'2700'` / `'3000'` / `'4000'` |
| `f.wood` | дерево `oak` / `walnut` / `black` |
| `f.zone` | зона `ceiling` / `wall` / `floor` / `table` |
| `FURN[id].ab` | вклад мебели в потери (0–8%, зеркало −3%) |
| `FURN[id].name` | имя предмета мебели |
| `Room.wallFinish` | `light` / `medium` / `dark` |
| `Room.ceilingH` | высота потолка, м |
| `baseLm(rt, room)` / `fxLm(fx)` | норма и факт люмен |
| `rt.zones` | доступные зоны в комнате |

## 0.1. Хелперы (новые в `engine/i18n.ts`)

| Функция | Что делает | Примеры |
|---|---|---|
| `kindWord(type, count)` | склонение типа светильника | 1: «люстра/торшер/настольная/спот/бра» · 2-4: «люстры/торшера/настольные/спота/бра» · 5+: «люстр/торшеров/настольных/спотов/бра» |
| `woodWord(wood, count)` | склонение породы | 1: «дуб/орех/чёрный дуб» · 2-4: «дуба/ореха/чёрных дуба» · 5+: «дубов/орехов/чёрных дубов» |
| `numWord(n)` | 1–4 буквой, 5+ цифрой | 1→«один», 2→«два», 3→«три», 4→«четыре», 5+→«5» |
| `joinList(arr)` | связка через запятую и «и» | `['люстры', 'бра', 'торшеров']` → «люстры, бра и торшеров» |
| `domKind(fixtures)` | имя доминанта по lm + размер | `[{m: rotor_1000}]` → «огромная люстра» |
| `secondaryList(fixtures, excludeId)` | список остальных | → «два торшера и пара бра» |

> ⚠️ В текстах **никогда** не используем `collection` («Rotor», «Rotor X», «Rotor Elliptical») и `name` модели («Rotor 1000»). Только `type` + `chip` (размер).

## 1. КАРТОЧКА «Где свет»

**Когда показывается:** всегда (если есть светильники).
**Chip:** `{доминантная зона} {pct}%` — нижний регистр: «потолок 55%», «стены 60%», «пол 40%», «мебель 30%».

### 1.1. Триггер по `scenePlace`

- `glade` (потолок ≥60%)
- `grove` (25–60%)
- `thicket` (≤25%)

### 1.2. Структура

```
{лид} {доминант}. {вторичные} {роль вторичных}. {обучение}
```

### 1.3. Банки

**Лид + доминант (по `place`):**
- glade:
  - «Главный свет — от {dom} сверху.»
  - «Сверху держит {dom}.»
  - «Свет идёт от {dom} сверху.»
- grove:
  - «{dom} светит сверху,»
  - «Сверху держит {dom},»
- thicket:
  - «Потолок тихий — свет держат {secondary}.»
  - «Свет ушёл вниз — к {secondary}.»

**Вторичные:** список из `joinList(secondaryList(...))`. Примеры:
- «два торшера и пара бра»
- «бра и настольная»
- «торшер»

**Роль вторичных (по `place`):**
- glade: «добирают объём по краям» · «подсвечивают зоны» · «снимают тени по углам»
- grove: «работают снизу — свет ложится ровно» · «подсвечивают комнату по краям»
- thicket: «(уже встроено в лид)»

**Обучение (по `place`):**
- glade:
  - «Один потолочный задаёт простой контур, а нижние источники добавляют объём.»
  - «Когда свет идёт и сверху, и снизу — он мягче, чем один источник из одной точки.»
- grove:
  - «Свет идёт со всех сторон — самое спокойное распределение для жилой комнаты.»
  - «И сверху, и снизу — в комнате нет тёмных островков, всё освещено ровно.»
- thicket:
  - «Свет снизу не бьёт в глаза — комната располагает к покою. Для работы за столом добавьте сверху.»
  - «Низкий свет хорош для тихого вечера — для уроков или готовки добавьте потолочный.»

### 1.4. Примеры сборки

| Сборка | Текст |
|---|---|
| 1× rotor_1000 + 2× бра + 2× торшер + 1× настольная (потолок 55%) | «Главный свет — от огромной люстры сверху. Два торшера и пара бра работают снизу — свет ложится ровно. Свет идёт со всех сторон — самое спокойное распределение для жилой комнаты.» |
| 1× rotor_m (потолок 100%) | «Главный свет — от средней люстры сверху. Один потолочный задаёт простой контур, нижние источники добавили бы объём.» |
| 2× бра + 1× торшер (потолок 0%) | «Свет ушёл вниз — к двум бра и торшеру. Низкий свет хорош для тихого вечера — для уроков или готовки добавьте потолочный.» |

## 2. КАРТОЧКА «Сколько света»

**Когда показывается:** всегда (если есть светильники).
**Chip:** `{ratio×100%}` — «66%», «105%», «145%».

### 2.1. Триггер по `brightLevel` (новые пороги: 0.8 / 1.2)

- `dim` (`<0.8`)
- `norm` (`0.8–1.2`)
- `full` (`≥1.2`)

### 2.2. Структура

```
{нормa} лм — {why_room}. У вас {actual} лм — {вердикт}. {обучение}
```

### 2.3. Банки

**Норма + why_room (per `RoomTypeId`):**
- kitchen: «Кухне {area} м² нужно ~{base} лм — там готовка и нож.»
- bedroom: «Спальне {area} м² нужно ~{base} лм — здесь читают и отдыхают.»
- living: «Гостиной {area} м² нужно ~{base} лм — для гостей и для вечера.»
- office: «Кабинету {area} м² нужно ~{base} лм — за столом работают с документами.»
- kids: «Детской {area} м² нужно ~{base} лм — за уроками глазам нужно много света.»
- hallway: «Коридору {area} м² нужно ~{base} лм — здесь проходят, не задерживаются.»
- bathroom: «Ванной {area} м² нужно ~{base} лм — у зеркала нужно много света.»
- stairs: «Лестнице {area} м² нужно ~{base} лм — важно видеть, куда ставить ногу.»

**Вердикт (по `brightLevel`):**
- dim: «у вас {actual} — на {pct}% от нормы, маловато» · «у вас {actual} — света не хватает»
- norm: «у вас {actual} — попадаете в норму» · «у вас {actual} — то, что нужно» · «{actual} — норма с лёгким запасом»
- full: «у вас {actual} — света с запасом» · «{actual} — больше нормы»

**Обучение (per_room × brightLevel; есть универсальные):**
- dim:
  - «На вечер хватит, для дел добавьте света.»
  - «Поставьте ещё один источник — света станет достаточно.»
  - kitchen+dim: «За готовкой будет темновато — добавьте свет над рабочей зоной.»
  - bedroom+dim: «Для отдыха достаточно, для чтения мало.»
  - office+dim: «Для бумаги мало — глаза устанут.»
- norm:
  - «Захотите вечером мягче — поможет диммер.»
  - living+norm: «Этого хватит и на гостей, и на тихий вечер.»
  - bedroom+norm: «Хватает на чтение, не давит на сон.»
  - kitchen+norm: «С готовкой справится, и с утра не режет глаза.»
  - office+norm: «За документами видно, к концу дня глаза не устанут.»
- full:
  - «Глазам этого даже много — поставьте диммер, будете крутить под настроение.»
  - «Поставьте диммер — приглушите вечером.»

### 2.4. Примеры сборки

| Сборка | Текст |
|---|---|
| Гостиная 25 м², 10 700 лм (норма 10 200, 105%) | «Гостиной 25 м² нужно ~10 200 лм — для гостей и для вечера. У вас 10 700 — попадаете в норму. Этого хватит и на гостей, и на тихий вечер.» |
| Спальня 16 м², 4 000 лм (норма 5 800, 69%) | «Спальне 16 м² нужно ~5 800 лм — здесь читают и отдыхают. У вас 4 000 — на 69% от нормы, маловато. Для отдыха достаточно, для чтения мало.» |
| Кабинет 12 м², 9 200 лм (норма 6 500, 142%) | «Кабинету 12 м² нужно ~6 500 лм — за столом работают с документами. У вас 9 200 — больше нормы. Поставьте диммер — приглушите вечером.» |

## 3. КАРТОЧКА «Стены»

**Когда показывается:** только если `wallFinish !== 'medium'`.
**Chip:** «светлые» / «тёмные» (для `light` / `dark`).

### 3.1. Банки

**light:**
- «Светлые стены отдают часть света обратно в комнату — норма ниже расчётной на ~10%.»
- «От светлых стен свет отскакивает в комнату. Светильников нужно на ~10% меньше.»
- «Стены работают на вас: возвращают свет. Снимаем с нормы ~10%.»

**dark:**
- «Тёмные стены забирают часть света — норму поднимаем на ~15%.»
- «Свет глохнет в тёмных стенах. Светильников нужно на ~15% больше.»
- «Тёмные стены съедают часть света. Добавляем ~15% к норме комнаты.»

> Для `medium` карточка **скрыта** — нечего сказать, не тратим место.

## 4. КАРТОЧКА «Дерево»

**Когда показывается:** всегда (если есть светильники).
**Chip:** «{n} {woodWord(n)}» через «·» по убыванию: «6 орехов · 3 дуба», «4 ореха», «1 чёрный дуб».

### 4.1. Триггер

- одна порода
- две или три породы

### 4.2. Структура

**Одна порода:**
```
{характеристика}. {обучение}
```

**Смесь:**
```
{w1} у {models1} — {char1}. {w2} у {models2} — {char2}. {обучение}
```

### 4.3. Банки

**Характеристика породы:**
- oak: «Дуб — светлый, тёплый.» / «Светлое дерево, тёплый тон.» / «Дуб — самый светлый из ламелей, спокойный тон.»
- walnut: «Орех — глубокий, тёплый.» / «Тёмное тёплое дерево.» / «Орех — глубокий тон, мягкая теплота.»
- black: «Чёрный дуб — строгий, плотный.» / «Самое тёмное дерево из трёх — строгий характер.»

**Сравнение для смеси (одно из двух):**
- «{w1} {где} — {char1}. {w2} {где} — {char2}.»
- «{w1} {где}, {w2} {где}. Свет разной плотности — у комнаты появляется объём.»

`{где}` собирается из `secondaryByWood(fixtures, wood)`:
- группировка по `wood`, для каждой — `kindWord` + список zones/типов
- примеры: «у люстры и торшеров» · «у бра и настольной» · «у одной люстры»

**Обучение (универсально):**
- «Ламели пропускают около двух третей света, и он становится мягким.»
- «Свет проходит сквозь ламели, и резкость уходит — комната получает мягкое освещение.»
- «Около трети света остаётся в дереве — так получается настроение живого леса.»

### 4.4. Примеры сборки

| Сборка | Chip | Текст |
|---|---|---|
| 6× орех (люстра+торшер) + 3× дуб (бра+настольная) | «6 орехов · 3 дуба» | «Орех у люстры и торшеров — глубокий, тёплый. Дуб у бра и настольной — светлый тон. Около трети света остаётся в дереве — так получается настроение живого леса.» |
| 4× орех (1 люстра) | «4 ореха» | «Везде орех — глубокий, тёплый. Ламели пропускают около двух третей света, и он становится мягким.» |
| 1× чёрный дуб (1 люстра) | «1 чёрный дуб» | «Чёрный дуб — строгий, плотный. Свет проходит сквозь ламели, и резкость уходит — комната получает мягкое освещение.» |

## 5. КАРТОЧКА «Мебель»

**Когда показывается:** только если `furniture.length > 0`.
**Chip:** `−{furnPct}%` (например «−20%») или «зеркало» (если в комнате только зеркало с `ab<0`).

### 5.1. Структура

```
{топ мебели с %}. {последствие}. {совет — условно}.
```

### 5.2. Сборка топ-мебели

- Сортируем по `FURN[id].ab` (по убыванию).
- Топ-2 самых тяжёлых: «{a} и {b} забирают по {pct}%».
- Хвост (>2 предметов): «остальная мебель — ещё {rest}%».
- Зеркало (если есть): «Зеркало возвращает 3% света — немного помогает.»

### 5.3. Банки

**Топ:**
- 1 предмет: «{name} забирает {pct}% света.»
- 2 предмета: «{a} и {b} забирают по {pct}% каждый.»
- 3+: «{a} и {b} забирают по {pcts}%, остальная мебель — ещё {rest}%.»

**Последствие (по `furnPct`):**
- ≤5%: «Тени почти не заметны.»
- ≤12%: «Углы у мебели в тени, но без потерь сцены.»
- >12%: «Углы у {top} в тени.» / «Тени собираются у {top}.»

### 5.4. Совет — НА ОСНОВЕ ПУСТЫХ ЗОН И КОНКРЕТНОЙ МЕБЕЛИ

Контекст:
- `available = rt.zones` — доступные зоны в комнате
- `used = Set(f.zone)` — занятые зоны
- `empty = available − used` — реально пустые зоны
- топовая мебель → подсказка, где источник пригодился бы

**Таблица привязки мебели к источнику (`FURN_HINT`):**

| Мебель (FurnId) | Зона | Где поставить | Польза |
|---|---|---|---|
| sofa (Диван) | floor | у дивана | подсветит этот угол |
| armchair (Кресло) | floor | у кресла | для чтения |
| bed (Кровать) | wall | у изголовья | снимет тень с подушки |
| bookshelf (Стеллаж) | wall | у стеллажа | добавит света в углы |
| wardrobe (Шкаф) | wall | у шкафа | осветит дверцы |
| dresser (Комод) | wall | над комодом | подсветит верх |
| desk (Стол) | table | на стол | уберёт тень от работы |
| dtable (Обед. стол) | ceiling | над столом | подсветит еду |
| kitchen_set (Кух. гарнитур) | wall | над столешницей | поможет с готовкой |
| tv_stand (ТВ-тумба) | wall | по бокам экрана | мягкий фон за телевизором |
| nightstand (Тумбочка) | table | на тумбочку | для чтения перед сном |

**Алгоритм совета:**

1. Найти топ-1 мебели по `ab`.
2. Взять `hint = FURN_HINT[topId]`.
3. Если `hint` нет → второй по тяжести и т.д.
4. Если `hint.zone` есть в комнате И пуста (никакого источника туда не подключено) → совет **«Добавьте {kind} {где} — {польза}.»**
5. Если `hint.zone` есть И занята → совет **«{Kind} {где} уже {делает своё дело-фраза}.»**
6. Если `hint.zone` нет в комнате → пропустить совет (или дать общий: «Углы остаются в тени.»)

**Банки «добавить»:**
- «Добавьте {kind} {где} — {польза}.»
- «{Kind} {где} {польза}.»
- «{Kind} {где} — лучшая пара к этой мебели.»

**Банки «уже стоит»:**
- «{Kind} {где} это уже компенсируют.»
- «{Kind} {где} делают своё дело — комната дышит.»
- «{Kind} {где} уже выравнивают тени.»

**Зеркало (если в списке `furniture`):**
- «Зеркало возвращает 3% света — немного помогает.»
- «Зеркало в комнате — отражает часть света обратно.»

### 5.5. Примеры сборки

| Мебель | Светильники / зоны | Chip | Текст |
|---|---|---|---|
| Диван, Стеллаж, Кресло, Журн. стол, ТВ-тумба | 2 бра в wall, 2 торшера в floor, 1 настольная в table | «−20%» | «Диван и стеллаж забирают по 6% каждый, остальная мебель — ещё 8%. Углы у стеллажа в тени. Бра у стеллажа делают своё дело — комната дышит.» |
| Кровать, Шкаф, Тумбочка | пусто в wall, ничего | «−17%» | «Кровать и шкаф забирают по 7–8%, тумбочка — ещё 2%. Тени собираются у шкафа. Добавьте бра у изголовья — снимет тень с подушки.» |
| Диван, ТВ-тумба | торшер в floor занят, wall пустой | «−9%» | «Диван забирает 6%, ТВ-тумба — ещё 3%. Углы у мебели в тени, но без потерь сцены. Бра по бокам экрана — мягкий фон за телевизором.» |
| Зеркало (в ванной) | свет у зеркала | «зеркало» | «Зеркало возвращает 3% света — немного помогает.» |

## 6. КАРТОЧКА «Оттенок света»

**Когда показывается:** всегда (если есть светильники).
**Chip:**
- одна температура: «2700K» / «3000K» / «4000K»
- смесь: перечисление через «·»: «2700K · 4000K»

### 6.1. Структура

**Одна температура:**
```
{характеристика}. {обучение/возможность}
```

**Смесь:**
```
{группа1: модели} светят {K1} — {назначение}. {группа2: модели} — {K2}, {назначение}. {обучение}
```

### 6.2. Банки

**Одна температура:**
- 2700K (тёплый):
  - «Везде тёплый 2700K — комната вечерняя.»
  - «Тёплый 2700K — для покоя. Для работы за столом маловат.»
  - «2700K — самый тёплый из трёх. Тёплый вечер по умолчанию.»
- 3000K (тёплый-нейтр.):
  - «Тёплый-нейтр. 3000K — компромисс. И поработать можно, и расслабиться.»
  - «3000K — между уютом и ясностью.»
- 4000K (рабочий):
  - «Везде 4000K — получается полноценный рабочий свет. Видно всё чётко.»
  - «4000K — нейтральный, без тёплого оттенка. Видно текстуры и цвета как есть.»

**Смесь (структура с подстановкой):**
- «{warm_models} светят {warm_K} — это вечерний свет. {clear_models} — {clear_K}, получается полноценный рабочий свет.»
- «{warm_models} на {warm_K}, {clear_models} — на {clear_K}. Два режима в одной комнате.»

`{warm_models}`, `{clear_models}` — группировка fixtures по `f.opts.btemp` → `kindWord` + zone hints:

## 7. КАРТОЧКА «Потолок и подвес»

**Когда показывается:** только если высота `> 2.7` ИЛИ есть подвес ИЛИ высота `< 2.7`. Стандартный 2.7 без подвеса → **скрыть**.
**Chip:** `{h} м` + ` · подвес` если есть.

### 7.1. Структура

```
{состояние потолка}. {роль подвеса/последствие}. {обучение опционально}
```

### 7.2. Банки

**Высота:**
- h > 3.0: «Потолок {h} м — высокий.»
- h ≈ 3.0: «Потолок 3 м — выше обычного.»
- h ≈ 2.7: (только если есть подвес) «Стандартные {h} м.»
- h < 2.7: «Потолок {h} м — невысокий.»

**Роль подвеса:**
- has pendant + dom (название доминантной потолочной):
  - «Подвес опускает {dom} ближе к комнате — возвращает около 6% света.»
  - «{dom} на подвесе — на 40 см ближе к работе, чем без него.»
- no pendant:
  - «Без подвеса источники остаются у потолка — свету идти дальше.» (если h > 2.7)
  - «Без подвеса источник близко к ламелям — следите, чтобы не били в глаза.» (если h < 2.7)

**Обучение (опционально):**
- h > 3 без подвеса: «Если поставить подвес, света станет заметно больше.»
- h < 2.7: «При низком потолке свет концентрируется — переборщить здесь легко.»

### 7.3. Примеры сборки

| Высота | Подвес | Chip | Текст |
|---|---|---|---|
| 3.0 м | да | «3 м · подвес» | «Потолок 3 м — выше обычного. Подвес опускает огромную люстру ближе к комнате — возвращает около 6% света.» |
| 3.2 м | нет | «3.2 м» | «Потолок 3.2 м — высокий. Без подвеса источники остаются у потолка — свету идти дальше. Если поставить подвес, света станет заметно больше.» |
| 2.7 м | да | «2.7 м · подвес» | «Стандартные 2.7 м. Подвес опускает люстру на 40 см ближе к работе.» |

## 4. Тон под состояние

| Состояние | Применение |

- dim
